# PINNED, not ranged. Why: Streamlit Cloud re-resolves dependencies on every
# redeploy, so with `>=` ranges two deploys of the SAME commit can run on
# DIFFERENT numpy/scipy/streamlit — a redeploy on a bad upstream release day
# becomes an outage you didn't cause and can't bisect. Exact pins make every
# deploy reproducible: same commit in, same environment out.
#
# The == pins below are the exact versions the full test suite (1111 tests)
# passed against on 2026-07-13. To upgrade: bump the pin on a branch, let CI
# run the suite against it, merge on green. Never edit pins directly on main.
#
# Place this file at the REPOSITORY ROOT (next to the folder that holds
# streamlit_app.py), which is where Streamlit Cloud looks for it.
